# shiva-playground

Welcome to **shiva-playground**, a personal GitHub organization used as a space for experimentation, learning, and building.

This organization is where I explore new technologies, create personal projects, and improve my software engineering workflow through hands-on practice.

Most projects in this organization will also be used to practice effective AI-assisted development, especially coding workflows with AI agents such as Codex.

## Purpose

The goal of this organization is to provide a dedicated place for:

- Experimenting with new programming languages, frameworks, libraries, and tools
- Building personal projects from idea to implementation
- Practicing GitHub workflows, automation, and CI/CD pipelines
- Improving project structure, documentation, and maintainability
- Learning better approaches to issue tracking, planning, and project management
- Exploring best practices for coding with AI agents, including planning, code review, testing, and documentation

## What You Will Find Here

Repositories in this organization may include:

- Proofs of concept
- Small applications and utilities
- Backend, frontend, and full-stack experiments
- DevOps and automation examples
- CI/CD workflow experiments
- Project templates and starter setups
- Personal learning projects

Some repositories may be polished and documented, while others may be experimental or work in progress.

## Focus Areas

This organization is especially focused on improving skills in:

- Software design and architecture
- Modern development tools and frameworks
- GitHub Actions and CI/CD automation
- Testing, linting, and code quality workflows
- Repository organization and documentation
- Agile planning and project management practices
- AI-assisted software development with tools such as Codex

## Philosophy

The main idea behind **shiva-playground** is simple: learn by building.

Each repository is an opportunity to test an idea, try a new technology, improve an existing workflow, or practice turning concepts into working software.

AI agents are treated as development collaborators: useful for accelerating implementation, reviewing ideas, improving documentation, and strengthening workflows, while keeping code quality, verification, and maintainability as the priority.

## Status

This is a personal playground, so projects may evolve, change direction, or be archived as learning goals are completed.
